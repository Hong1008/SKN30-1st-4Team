# 데이터베이스 설계서 (Database Architecture)

> **전기차 충전 인프라 SOS 대시보드**의 데이터 저장소 구조와 설계를 기술합니다.
> 본 프로젝트는 연도별 통계 데이터와 지역 메타데이터를 효율적으로 관리하기 위해 관계형 모델을 채택했습니다.

---

## 1. 개요
본 시스템의 데이터베이스는 전기차 등록 현황, 충전기 보급 현황, 그리고 사용자 민원(FAQ) 데이터를 통합 관리합니다. 초기 단계에서는 CSV 기반 파이프라인으로 동작하며, 대규모 확정 및 영구 보관이 필요한 시점에 MySQL 서버로 이관될 수 있도록 설계되었습니다.

---

## 2. ER 도식 (ER Diagram)

![ERD](image.png)

### 관계 정의
- **`year` ↔ `stat_ev`**: 1:N 관계. 한 연도에 여러 지역의 통계 데이터가 존재합니다.
- **`year` ↔ `qna`**: 1:N 관계. 특정 연도에 발생한 여러 민원/FAQ 항목을 관리합니다.
- **`region` ↔ `stat_ev`**: 1:N 관계. 한 지역에 대해 여러 연도의 히스토리 통계가 쌓입니다.
- **`stat_ev`**: 복합 기본키(`year`, `region_name`)를 통해 연도별 지역 데이터의 유일성을 보장합니다.

---

## 3. 상세 테이블 정의

### 3.1 `year` (연도 마스터)
데이터의 기준이 되는 연도 정보를 관리합니다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|:---|:---|:---|:---|
| `year` | SMALLINT | PK | 기준 연도 (예: 2020, 2021 ...) |

### 3.2 `region` (지역 마스터)
전국 8개 광역 지자체의 지리적/공간적 메타데이터를 관리합니다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|:---|:---|:---|:---|
| `region_name` | VARCHAR(50) | PK | 표준 지역 명칭 (서울특별시, 경기도 등) |
| `lat` | DOUBLE | - | 해당 지역의 중심 위도 |
| `lon` | DOUBLE | - | 해당 지역의 중심 경도 |
| `area` | DOUBLE | - | 지역 면적 (km²) — 불편지수 산출 활용 |

### 3.3 `stat_ev` (전기차 통계)
연도별/지역별 핵심 수치 데이터를 저장하는 메인 팩트 테이블입니다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|:---|:---|:---|:---|
| `year` | SMALLINT | PK, FK | `year.year` 참조 |
| `region_name` | VARCHAR(50) | PK, FK | `region.region_name` 참조 |
| `total_ev_count` | INT | - | 총 전기차 등록 수 |
| `total_charger_count` | INT | - | 총 충전기 보급 수 |

### 3.4 `qna` (민원/FAQ)
수집된 사용자 민원 데이터 및 FAQ 내용을 관리합니다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|:---|:---|:---|:---|
| `qna_id` | INT | PK | 고유 식별자 |
| `title` | VARCHAR(500) | - | 민원 또는 FAQ 제목 |
| `year` | SMALLINT | FK | 발생 연도 (문맥 분석 기반) |

## 4. 요구사항 기반 설계 반영

본 설계는 [요구사항 명세서](requirements.md)의 다음 항목들을 충실히 반영하고 있습니다.

1. **광역별 연도별 데이터 관리**: `year`와 `region`을 PK로 결합한 `stat_ev` 테이블을 통해 2020~2025년 통계 조회 요구사항을 충족합니다.
2. **불편지수 산출 지원**: `region.area` 컬럼을 통해 공간 기반 지수(TCII) 계산에 필요한 면적 데이터를 제공합니다.
3. **지리 정보 시각화**: `region.lat`, `region.lon`을 통해 지도 마커 및 핫스팟 시각화에 필요한 좌표 데이터를 관리합니다.
4. **민원 키워드 분석**: `qna` 테이블을 통해 크롤링된 텍스트 데이터를 저장하고 연도별 키워드 추출(WordCloud)을 지원합니다.
